# 큐텐 경쟁사 비교분석 도구 — 개발 명세서

작성일: 2026-06-05 · 상태: 설계 (방법론 확정분 기반)
목적: 올리브영 비딩 1회성이 아닌, **재사용 가능한 경쟁사 분석 파이프라인**으로 개발.
기반: 기존 `qoo10_scraper` 확장 (Playwright + SQLite + pandas)

---

## 1. 시스템 개요

```
[shops_groups.py 그룹 설정]
        │
[1] 수집기 (collector)          ── 샵/상품/이미지 일일 스냅샷 → SQLite + images/
        │
[2] 스코어링 엔진 (scoring)      ── 정량 지표 계산 + 정성 루브릭 채점 병합
        │
[3] 리포트 생성기 (report)       ── 스코어카드 xlsx · 포지셔닝 맵 png · 보고서 pptx
```

실행 예:
```bash
python collect.py --group skincare        # 그룹 단위 수집 (일 1회 cron)
python score.py   --group skincare        # 지표 계산 + 루브릭 병합
python report.py  --group all --pptx      # 보고서 생성
```

## 3. 수집기 명세 (`collect.py`)

### 3-1. 샵 레벨 스냅샷 (신규 테이블 `shop_snapshots`)

| 필드 | 추출 위치/방법 |
|---|---|
| seller_grade | 샵 헤더 POWER 아이콘 alt/이미지명 |
| satisfaction_pct | 헤더 "94%" 텍스트 |
| follower_count | "フォロワー 75,422" 파싱 |
| product_count | "全ての商品 (24)" 파싱 |
| coupon_count / coupon_max_off / coupon_min_spend | 쿠폰 다운로드 블록 |
| timesale_count / timesale_avg_off | "タイムセール n%↓" 집계 |
| megawari_sku_count | 상품명 메가와리 키워드(メガポ/メガ割) 매칭 수 |

### 3-2. 상품 레벨 (기존 `snapshots` 재사용 + 파생 컬럼)

기존 수집(코드·이름·가격·정가·리뷰수·평점·판매배지)에 분류 파생 추가:

| 파생 필드 | 규칙 |
|---|---|
| is_set | 상품명에 セット/企画/SET 포함 |
| is_jp_limited | 日本限定/日本先行 포함 |
| discount_pct | (list_price − price) / list_price |

### 3-3. 이미지 수집 (신규 `images.py`)

- 메인 썸네일: 상품 목록의 대표 이미지 URL → `images/{shop_id}/thumb_{goods_code}.jpg`
- 샵 메인 배너: 상단 배너 img → `images/{shop_id}/banner_{n}.jpg`
- 전체 페이지 캡처: Playwright `page.screenshot(full_page=True)` → `images/{shop_id}/shoptop_{date}.png` (루브릭 채점·PPTX 갤러리용)
- 메타는 `image_assets` 테이블에 기록 (shop_id, type, path, captured_at)

### 3-4. 수집 매너

- 일 1회, 샵 간 5초 이상 대기. 실패 샵은 스킵 후 로그 (전체 중단 금지 — run.py 패턴 동일).

## 4. DB 스키마 추가분

```sql
CREATE TABLE IF NOT EXISTS shop_snapshots (
    shop_id          TEXT,
    snap_date        TEXT,
    seller_grade     TEXT,
    satisfaction_pct REAL,
    follower_count   INTEGER,
    product_count    INTEGER,
    coupon_count     INTEGER,
    coupon_max_off   TEXT,
    timesale_count   INTEGER,
    timesale_avg_off REAL,
    megawari_sku_count INTEGER,
    PRIMARY KEY (shop_id, snap_date)
);

CREATE TABLE IF NOT EXISTS image_assets (
    shop_id     TEXT,
    asset_type  TEXT,     -- 'thumb' | 'banner' | 'shoptop'
    goods_code  TEXT,     -- thumb일 때만
    path        TEXT,
    captured_at TEXT,
    PRIMARY KEY (shop_id, asset_type, path)
);

CREATE TABLE IF NOT EXISTS design_scores (   -- 정성 루브릭 (수기 입력)
    shop_id     TEXT,
    scored_at   TEXT,
    thumbnail   REAL,     -- 가중치 25%
    shop_main   REAL,     -- 20%
    localization REAL,    -- 20%
    promo_design REAL,    -- 20%
    consistency REAL,     -- 15%
    scorer      TEXT,
    note        TEXT,
    PRIMARY KEY (shop_id, scored_at, scorer)
);
```

## 5. 스코어링 엔진 (`score.py`)

### 5-1. 정량 지표 (자동 계산)

| 지표 | 산식 |
|---|---|
| 가격 포지션 | 전 상품가 min / median / max |
| 리뷰 볼륨 | Σ review_count (판매력 대리지표) |
| Top5 파워 | 리뷰수 상위 5개 합 / 전체 합 (히어로 의존도) |
| 세트 비중 | is_set 상품 수 / 전체 |
| 일본 한정 기획력 | is_jp_limited 수 |
| 추정 매출 | 스냅샷 2일↑ 시 리뷰 차분 ÷ 0.30 × 가격 (estimator.py 재사용) |
| 프로모션 강도 | coupon_count + timesale_count 표준화 점수 |

### 5-2. 디자인 루브릭 (수기 채점 → 자동 병합)

- 채점 입력: `design_scores` 테이블 또는 `rubric_{group}.csv` 업로드
- 5점 척도 기준표(1/3/5점 앵커)는 방법론 문서 기준 유지:
  - 썸네일(25%): 5점 = 소셜프루프 배지 + 베네핏 시각화 + 톤 통일
  - 샵 메인(20%): 5점 = 스토리·기획전·랭킹·이벤트 구좌 체계화
  - 현지화(20%): 5점 = TPO 컷 + UGC + 일본 정서 카피
  - 프로모션 설계(20%): 5점 = 쿠폰·타임세일·세트 유기적 동선
  - 일관성(15%): 5점 = 컬러·폰트·레이아웃 가이드 일관
- 디자인 총점 = Σ(점수 × 가중치), 그룹 내 순위 산출

### 5-3. 출력

- `out/scorecard_{group}.xlsx`: 샵 × 전 지표 매트릭스 (자사 행 하이라이트, 1위 대비 갭 컬럼)

## 6. 리포트 생성기 (`report.py`)

| 산출물 | 구현 |
|---|---|
| 포지셔닝 맵 ① 가격(중앙값)×리뷰볼륨 ② 팔로워×만족도 | matplotlib scatter, 자사 강조 마커 → png |
| 디자인 벤치마크 갤러리 | image_assets에서 그룹 Best vs 자사 썸네일 나란히 배치 |
| 갭 분석 표 | 지표별 (자사 − 그룹 1위) 정렬 → 개선 우선순위 |
| PPTX 보고서 | python-pptx — ①표지 ②요약 ③스코어카드 ④포지셔닝 맵 ⑤갤러리 ⑥갭 분석→개선 과제 ⑦부록(원본 데이터) |

PPTX 템플릿은 16:9, 그룹당 1부 또는 통합 1부 옵션 (`--pptx --merge`).

## 8. 한계·주의

- 리뷰 기반 매출 추정은 ±30~50% 오차 — 절대값이 아닌 **샵 간 상대 비교** 용도.
- Qoo10 DOM 변경 시 셀렉터 보정 필요 (`--debug` 우선 실행).
- 디자인 루브릭은 수기 채점 단계가 남음 — 채점 2인 교차로 편향 완화.
- 수집 데이터·이미지는 내부 분석 용도로만 사용 (저작권·약관 고려).
